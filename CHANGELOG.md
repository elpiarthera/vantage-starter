### Fixed (2026-07-17 — give Jest/Vitest/Playwright non-overlapping, derived territory; the false-green ratio is gone)

**Correction to the entry below this one.** The first pass of this fix called `pnpm test`'s "94/94 passed" printed above "91 failed, 4 passed" (Test Suites) an out-of-scope, pre-existing config quirk. It is not out of scope and it is not someone else's runner: every one of those 95 suites is this repo's own, and "94/94" reading as 100% while 96% of the suite never started IS the false green this task exists to close — not a footnote to it.

**Root cause, measured:** `jest.config.ts`'s `testMatch` (`**/__tests__/**/*.[jt]s?(x)` + `**/?(*.)+(spec|test).[jt]s?(x)`) matched every `.test.*`/`.spec.*` file in the repo with no territory boundary at all — 6 Playwright specs under `e2e/` (using `@playwright/test`, which Jest's transform cannot parse) and 35 Vitest suites (`import ... from "vitest"`, same problem) all matched Jest's glob too. Jest reported each as a FAILED SUITE while its "Tests: N/N passed" line only ever counts tests inside suites that actually started — so the two lines told opposite stories and the wrong one was easy to read as green. `vitest.config.ts` had the mirror problem: no `test.include` at all, so a bare `vitest run` fell back to its own default glob and picked up the 3 real Jest suites plus 1 orphan file that called zero test-runner API.

- **Territory derived, never typed.** Added `scripts/derive-test-runner-ownership.js` — the single source of truth both `jest.config.ts` and `vitest.config.ts` now `require()`/`import` at config-load time, so ownership cannot drift between the two configs (they call the exact same function). Three rules, in order: (1) PLAYWRIGHT owns whatever directory `playwright.config.ts` itself declares as `testDir` — read out of that file with a regex, never retyped; (2) VITEST owns every remaining file whose own source contains `from "vitest"` — an inherent, checkable property of the file, not a directory guess; (3) JEST owns every remaining file with at least one `describe(`/`test(`/`it(` call (a real suite, relying on Jest's global-injection model). Anything matched by none of the three is an ORPHAN, reported by name, never silently caught by whichever glob still reaches it.
- **Honest ratio, measured per runner (suites on disk vs. started vs. passed):**
  - Jest: **4 on disk, 4 started, 4 passed** (94/94 tests). `pnpm test` now exits **0**.
  - Vitest: **39 on disk, 39 started**, 26 passed / 13 failed (18 individual test failures) — real, named, pre-existing-in-the-codebase failures now visible for the first time because these suites were never actually exercised end-to-end before: missing source files (`convex/voiceModels.ts`, `app/api/step1/generate-story/route.ts`, `convex/actions/videoAssembly.ts` no longer exist — dead tests pointing at removed/renamed product code), missing `POLAR_PRODUCT_*` env vars, and prompt-template assertions drifted from the current prompt copy.
